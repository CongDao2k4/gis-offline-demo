Chạy lại docker tileserver:

```bash
docker run --rm -it \
  -v "$PWD/output":/data \
  -v "$PWD/tileserver":/config \
  -p 8081:8080 \
  maptiler/tileserver-gl \
  --config /config/config.json
```

Sau đó mở `http://localhost:8081/`

sau đó chạy Spring boot: `./mvnw spring-boot:run`

Để cập nhật dump data mới vào PostgreDb. hãy đọc ![file chạy dump vào DB](../osm_postgre/guide_osm_postgre.md) với tên file đổi thành `data/hanoi.osm.pbf`
-> Tôi chọn chạy `scripts/03_import_postgis.sh` với tên `data/hanoi.osm.pbf`

### Nếu chạy tay thì làm như sau:

- Xóa schema cũ: tại bash: `psql -h localhost -U gisuser -d gis` 
- Liệt kê bảng: `\dt`

    thì kết quả kiểu:
    
    ```text
    planet_osm_line
    planet_osm_point
    planet_osm_polygon
    planet_osm_roads
    ...
    ```
  
- Xóa sạch: 
```sql
DROP TABLE IF EXISTS planet_osm_line CASCADE;
DROP TABLE IF EXISTS planet_osm_point CASCADE;
DROP TABLE IF EXISTS planet_osm_polygon CASCADE;
DROP TABLE IF EXISTS planet_osm_roads CASCADE;
DROP TABLE IF EXISTS planet_osm_nodes CASCADE;
DROP TABLE IF EXISTS planet_osm_rels CASCADE;
DROP TABLE IF EXISTS planet_osm_ways CASCADE; 
```

- Thoát ra: `\q`
- chạy lại:
```bash
export PGPASSWORD=gispassword

osm2pgsql \
  --create \
  --database gis \
  --username gisuser \
  --host localhost \
  --port 5432 \
  --slim \
  --hstore \
  data/hanoi.osm.pbf 
```


## Chạy Java trong WSL mới được:
```bash
sudo apt update
sudo apt install -y openjdk-21-jdk
```

Check lại các tool

```bash
java -version
docker --version
osmium --version
curl --version
```

Sau đó chạy: `./mvnw spring-boot:run`

Mở giao diện như bình thường: `http://localhost:8080/map/index.html`

Kiểm tra port: `docker ps` để tránh cổng 8091 bị chiếm.
